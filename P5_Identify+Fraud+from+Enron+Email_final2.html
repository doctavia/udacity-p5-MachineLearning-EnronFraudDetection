def featureFormat( dictionary, features, remove_NaN=True, remove_all_zeroes=False, remove_any_zeroes=False, sort_keys = 'C:/Users/Dewi Octavia/Documents/machinelearning_tools/python2_lesson14_keys.pkl'):
    """ convert dictionary to numpy array of features
        remove_NaN = True will convert "NaN" string to 0.0
        remove_all_zeroes = True will omit any data points for which
            all the features you seek are 0.0
        remove_any_zeroes = True will omit any data points for which
            any of the features you seek are 0.0
        sort_keys = True sorts keys by alphabetical order. Setting the value as
            a string opens the corresponding pickle file with a preset key
            order (this is used for Python 3 compatibility, and sort_keys
            should be left as False for the course mini-projects).
        NOTE: first feature is assumed to be 'poi' and is not checked for
            removal for zero or missing values.
    """


    return_list = []

    # Key order - first branch is for Python 3 compatibility on mini-projects,
    # second branch is for compatibility on final project.
    if isinstance(sort_keys, str):
        import pickle
        keys = pickle.load(open(sort_keys, "rb"))
    elif sort_keys:
        keys = sorted(dictionary.keys())
    else:
        keys = dictionary.keys()

    for key in keys:
        tmp_list = []
        for feature in features:
            #if feature != 'email_address':
            try:
                dictionary[key][feature]
            except KeyError:
                print "error: key ", feature, " not present"
                return
            value = dictionary[key][feature]
            if value=="NaN" and remove_NaN:
                value = 0
            tmp_list.append( float(value) )

        # Logic for deciding whether or not to add the data point.
        append = True
        # exclude 'poi' class as criteria.
        if features[0] == 'poi':
            test_list = tmp_list[1:]
        else:
            test_list = tmp_list
        ### if all features are zero and you want to remove
        ### data points that are all zero, do that here
        if remove_all_zeroes:
            append = False
            for item in test_list:
                if item != 0 and item != "NaN":
                    append = True
                    break
        ### if any features for a given data point are zero
        ### and you want to remove data points with any zeroes,
        ### handle that here
        if remove_any_zeroes:
            if 0 in test_list or "NaN" in test_list:
                append = False
        ### Append the data point if flagged for addition.
        if append:
            return_list.append( np.array(tmp_list) )

    return np.array(return_list)


def targetFeatureSplit( data ):
    """ 
        given a numpy array like the one returned from
        featureFormat, separate out the first feature
        and put it into its own list (this should be the 
        quantity you want to predict)

        return targets and features as separate lists

        (sklearn can generally handle both lists and numpy arrays as 
        input formats when training/predicting)
    """

    target = []
    features = []
    for item in data:
        target.append( item[0] )
        features.append( item[1:] )

    return target, features

def computeFraction( poi_messages, all_messages ):
    """ given a number messages to/from POI (numerator) 
        and number of all messages to/from a person (denominator),
        return the fraction of messages to/from that person
        that are from/to a POI
   """

    if poi_messages == 'NaN' or all_messages == 'NaN':
        fraction = 0
    else:
        fraction = float(poi_messages)/float(all_messages)

    return fraction

3. Pick and Tune an Algorithm¶

3.1 Pick an Algorithm¶

As Naive Bayes does not have parameters to tune, Decision Tree with second highest F1 score is selected as the final algorithm. Both algorithms met the precision and recall scores criteria of greater than 0.30.

F1 scoring metric is a better evaluation metric for this project because it measures accuracy using both precision and recall. F1 is the selected F-score subclass since algorithm passing criteria is having both precision and recall to be greater than 0.3 hence it is assumed that precision and recall are equally important. If the passing criteria was to weigh precision higher than recall or the other way around, F-score subclass of F0.5 or F2 would be chosen [https://en.wikipedia.org/wiki/F1_score].

The list and score summary of algorithms evaluated can be found in Feature Selection section.

3.2 Tune the Algorithm¶

Parameter tuning is a method of maximising algorithm performance by selecting the best parameters. The process in finding the optimum parameters is usually dependant on the data that the algorithms are deployed to, and requires a number of trials.

In this project, parameters tuning for each algorithms is done through GridSearchCV and it was cross validated as it goes. For the final algorithm, Decision Tree, the parameters tuned are as follows:

  1. criterion: ('gini', 'entropy')
  2. min_samples_split: [2, 5, 10, 15, 25, 50].

The parameters range was chosen based on their default values and some trials.

4. Validate and Evaluate¶

4.1 Validation and its Importance¶

Validation is the process of assessing how well the algorithm performs by separating data set into training set and test set. The algorithm is fitted on training set and the trained algorithm is then tested on test set to determine its performance. Cross validation is used to ensure the trained model generalizes well, without overfitting or underfitting. Cross validation performs the validation process multiple times, randomly splits data in each time.

In this project, 20 folds of stratified cross validation in random state was performed for grid search and a 1000 folds of stratified cross validation in random state was performed for evaluation. As the Enron dataset is a small and skewed dataset, cross validation is a better option.

4.2 Usage of Evaluation Metrics¶

As mentioned in previously, precision and recall are used as the evaluation metric in this project:

  • Precision measures the ratio of POIs classified correctly to total number of predicted POIs.
  • Recall measures the ratio of POIs classified correctly to total number of POIs.

Accuracy score measures the ratio of POIs and non-POIs classified correctly out of all data points. Enron dataset has 18 POIs and 128 non-POIs. Even if the identifier misses any POIs, the accuracy score would still be reasonably high hence accuracy score metric is not suitable for evaluating POI identifier.

In the chosen algorithm, F1 score is 0.382, and precision and recall scores are 0.437 and 0.339, respectively. This means 43.7% of flagged POIs is truly POIs and 33.9% of POIs are classified correctly out of total number of POIs, that is out of 18 POIs, 6 POIs are identified correctly.
